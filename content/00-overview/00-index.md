# Programming Concepts: The Beautiful Fundamentals

> *"Programs must be written for people to read, and only incidentally for machines to execute."*  
> ‚Äî Harold Abelson, Structure and Interpretation of Computer Programs

## üåü Why This Course Exists

In the age of AI-assisted coding, you might wonder: *Why learn these concepts when AI can generate code for me?*

The answer is simple yet profound: **These concepts don't just make you productive‚Äîthey transform how you think.**

This course teaches you the ideas that are:
- **Beautiful**: Elegant solutions that reveal deep truths about problem-solving
- **Lifechanging**: Patterns of thought that restructure how you approach complexity
- **Timeless**: Principles that transcend languages, frameworks, and yes, even AI tools
- **Practical**: Skills that let you architect, review, debug, and improve code‚Äîwhether written by humans or AI

### The Truth About AI-Assisted Development

AI is a powerful amplifier:
- **With understanding**: You architect systems, guide AI precisely, spot flaws instantly, and make wise trade-offs
- **Without understanding**: You copy-paste code you can't debug, miss security issues, create unmaintainable systems, and hit walls when things go wrong

This course gives you the **mental models** to be the architect, not just the typist.

## üìñ Modules Overview

### [Module 1: Type Systems](../01-type-systems/00-index.md)
*"Types are the algebra of programs"*

Learn how TypeScript's type system lets you encode invariants, catch errors at compile-time, and document your intent.

**Key Topics**: Type theory, generics, conditional types, type inference, branded types

### [Module 2: OOP Principles](../02-oop-principles/00-index.md)
*"Objects should be easy to use and hard to misuse"*

Master SOLID principles, understand composition over inheritance, and learn to model domains elegantly.

**Key Topics**: SOLID, encapsulation, polymorphism, composition, domain modeling

### [Module 3: Design Patterns](../03-design-patterns/00-index.md)
*"These are the building blocks of elegant solutions"*

Study the most important patterns that solve recurring problems.

**Key Topics**: Creational, structural, and behavioral patterns; pattern trade-offs

### [Module 4: Functional Programming](../04-functional-programming/00-index.md)
*"Purity is freedom from side effects and time"*

Experience the paradigm shift: immutability, pure functions, composition.

**Key Topics**: Immutability, pure functions, higher-order functions, functors, monads

### [Module 5: Async Programming](../05-async-programming/00-index.md)
*"Time is just another dimension to manage"*

Master promises, async/await, and concurrency patterns.

**Key Topics**: Promises, async/await, event loop, concurrency, error handling in async code

### [Module 6: Error Handling](../06-error-handling/00-index.md)
*"Errors are data, handle them gracefully"*

Learn to distinguish error types, implement robust error handling.

**Key Topics**: Result types, railway-oriented programming, validation, error boundaries

### [Module 7: Data Structures](../07-data-structures/00-index.md)
*"The right structure makes impossible problems trivial"*

Understand how data structures shape algorithms.

**Key Topics**: Trees, graphs, hash tables, complexity analysis, memory management

### [Module 8: Software Architecture](../08-architecture/00-index.md)
*"Architecture is about making change easy"*

Design systems that scale, change gracefully, and remain understandable.

**Key Topics**: Layered architecture, dependency injection, modular design, API design

### [Module 9: Testing & Quality](../09-testing/00-index.md)
*"Tests are specifications that are executable"*

Write tests that document behavior and catch regressions.

**Key Topics**: Unit testing, integration testing, TDD, test patterns, code quality

### [Module 10: Performance](../10-performance/00-index.md)
*"First make it work, then make it beautiful, then make it fast"*

Learn to measure, profile, and optimize.

**Key Topics**: Complexity analysis, profiling, optimization patterns, memory efficiency

### [Module 11: AI-Era Development](../11-ai-era-programming/00-index.md)
*"AI amplifies understanding; it doesn't replace it"*

Learn to work effectively with AI tools while maintaining craftsmanship.

**Key Topics**: AI-assisted development, reviewing generated code, architectural decisions

### Required
- **Node.js** 20+ (LTS version recommended)
- **npm** or **yarn** (comes with Node.js)
- **A code editor** (VS Code recommended)

### Initial Setup

```bash
npm install
```

This installs:
- TypeScript compiler
- Vitest (testing framework)

- Type definitions

### Verify Installation

```bash
# Type check all code
npm run type-check

# Run all tests
npm test

# Check linting
npm run lint

# Format code
npm run format
```

### Available Commands

| Command | Purpose |
|---------|---------|
| `npm test` | Run all tests |
| `npm run test:ui` | Run tests with UI |
| `npm run test:coverage` | Generate coverage report |
| `npm run type-check` | Check TypeScript types |
| `npm run lint` | Check code style |
| `npm run lint:fix` | Auto-fix linting issues |
| `npm run format` | Format all code |
